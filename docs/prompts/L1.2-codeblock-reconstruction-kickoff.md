## The hard problem (why this is heuristic — grounded in the real gold)

Inspect first — don't guess. The patterns in `~/data/vdocs/documents/gold/consolidated/DI/fm22_2dg/body.md`:

```bash
D=~/data/vdocs/documents/gold/consolidated/DI/fm22_2dg
sed -n '925,965p' "$D/body.md"     # Example 1 / Example 2: '^DD(...)=...' lines under '##### Example N'
grep -nE '^\*\*[SDWKNIFQXG] ' "$D/body.md" | head      # candidate standalone bold code statements
grep -nE '\*\*\\?\$ORDER\*\*|\*\*KILL\*\*|\*\*NEW\*\*' "$D/body.md" | head   # inline emphasis — MUST be left alone
```

Three things to recover, and one thing to **not** touch:

1. **Standalone bold code statements** → fence. A line whose *entire* content (ignoring the wrapping
   `**…**`) is an M statement, e.g. `**S DIC="^DIZ(662001,",L=0,BY=.01,...**`. Heuristic: after
   stripping bold markers, the line starts with an M command verb (`S SET D DO W WRITE K KILL N NEW
   I IF F FOR Q QUIT X XECUTE G GOTO H HANG M MERGE R READ`) or a global ref `^`, and is M-shaped
   (operators `= ^ $ _`), and is **not** predominantly natural-language prose.
2. **Escaped global listings** (plain, not bold) → fence. Lines like
   `^DD(filenumber,0,"ID","W1")=W " ",\$P(^(0),U,2)` — start with `^`, carry an `=` assignment and
   `\$`/`\_` markdown-escape artifacts. Group *consecutive* such lines into one block.
3. **Caption binding** (fixes defect D-2): a `##### Example N` / `Example N:` / `Figure N:` line
   immediately preceding a code region becomes the block's label (keep the heading/bold line right
   before the fence).
4. **DO NOT fence inline emphasis.** Prose lines like *"The **\$ORDER** function is used…"* are normal
   sentences with emphasized M tokens — leave them entirely unchanged. This is the main false-positive
   risk; bias toward leaving prose alone.

Also: **un-escape inside reconstructed blocks only** — `\$`→`$`, `\_`→`_`, `\*`→`*`, `\[`→`[`,
`\]`→`]`. Outside code, leave markdown escapes (they render fine as `$ORDER`).

## Design

- **Language tag:** ` ```mumps ` for code; ` ```console ` for a captured roll-and-scroll terminal
  session (device prompts / `Select …:` / user-typed responses interleaved with output). Default to
  `mumps`; use `console` only when terminal-session signals are clearly present (document the rule).
- **`manual-review` escape hatch (never silent):** a region the detector can't confidently classify
  (caption with no clearly-code lines after it; a line that's part code/part prose) is wrapped with
  `<!-- vdocs-codeblock: manual-review — <reason> -->` and left as-is, **counted in the report**. The
  plan's gate forbids silent drops (a flagged region reads as "needs a human," not "done").
- **Return a report** alongside the body: `{blocks_fenced, lines_unescaped, manual_review_regions,
  …}` so L1.4's driver / the gate can print "N blocks reconstructed, M flagged for manual review."
- **Pure + deterministic:** input string → output string + report; no clock, no I/O, no globals. Same
  input → identical output.
