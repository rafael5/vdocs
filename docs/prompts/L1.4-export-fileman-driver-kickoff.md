## The per-document pipeline (apply in this order)

Input bundle: `~/data/vdocs/documents/gold/consolidated/DI/<slug>/` → `body.md` + `tables/*.csv` +
sidecars. Use `Settings()` for lake paths.

1. **Parse** `kernel.frontmatter.parse(body)` → `(meta, body)`.
2. **Rewrite front matter** — `frontmatter_pure.rewrite_frontmatter(meta, slug=…, imported_by=…, imported_date=…)`. Add `track`/target into front matter if useful.
3. **Reconstruct code blocks** — `codeblock_pure.reconstruct(body)` (do this BEFORE tables/assets so escape-unescaping stays inside fences). Keep the report.
4. **Materialize tables** — for each `_[Table N …](tables/table-NN.csv)_` placeholder: read the CSV via `kernel.csv.read_rows`, `tables_pure.materialize(rows)`; replace the placeholder with the GFM table; if `kind == "reference"`, also write `data/<slug>-table-NN.yml` (the records) and **drift-check** the rendered table against the records.
5. **Resolve assets** — rewrite both `![](<sha>.ext)` AND raw `<img src="<sha>.png" …>` (the Developer's Guide uses raw `<img>`): copy the CAS bytes (`documents/assets/<sha>.<ext>`) into the topic's `media/` as `fig-NN.ext`, rewrite the ref to the relative `media/…` path. **Reuse `kernel/figures.py`** (built for rich-publication — read its API first; it already resolves sha→CAS bytes). Sequential `fig-NN` names are fine (alt-text backfill is a later quality pass).
6. **Resolve boilerplate** — `_[… — shared boilerplate](_shared/boilerplate/bp-<hash>.md)_` links point at `gold/_shared/boilerplate/bp-<hash>.md` in the lake. **Decision (recommend INLINE for L1):** inline the boilerplate prose so the topic is fully self-contained and raw-readable on github.com (strategy S5; resolves master-proposal Q3 toward self-contained for the imported baseline). `_includes/` single-sourcing can be an L3 refinement. Whatever you choose, leave **no** `bp-<hash>` placeholder behind.
7. **Emit** — `kernel.frontmatter.emit(new_meta, new_body)` → write to `<out-dir>/<track>/<file>.md`.

or document the deferral). The L1 structural gate (no dangling refs, no leftover placeholders,
schema-valid front matter) is the bar here.
